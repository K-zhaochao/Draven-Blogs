### 线程安全的问题

当多个线程操作同一个数据的时候就会出现问题。

#### 举例：

![[Draven_Note_Images/Pasted image 20260606155503.png]]
**问题1：** 相同的票出现多次
**问题2：** 出现了超出范围的票
原因是线程是有随机性的，当某个线程还未执行最终操作的时候，CPU的执行线程就会被其他线程抢占过去。

#### 同步代码块：

![[Draven_Note_Images/Java/23.多线程/23. 多线程_同步代码块：_12.png]]

``` java
// 用继承Thread的方式实现
public class MyThread extends Thread{  
    // 表示这个类所有的对象，都共享ticket数据  
    static int ticket = 0;  
  
    // 锁对象，一定要是唯一的  
    static Object lock = new Object();  
    public MyThread(String name) {  
        super(name);  
    }  
    @Override  
    public void run() {  
        while (true) {  
            synchronized(lock) {  
                if (ticket < 100) {  
                    try {  
                        Thread.sleep(100);  
                    } catch (InterruptedException e) {  
                        throw new RuntimeException(e);  
                    }  
  
                    ticket++;  
                    System.out.println(getName() + "正在售卖第" +  ticket + "张票！");  
                } else {  
                    break;  
                }  
            }  
        }  
  
    }  
}
```

``` java
// 用实现Runnable的方式来实现
public class MyRunnable implements Runnable {  
    // 表示这个类所有的对象，都共享ticket数据  
    int ticket = 0;  
    @Override  
    public void run() {  
        // 1.循环  
        while (true) {  
            // 2.同步代码块（同步方法）  
            synchronized (MyRunnable.class) {  
                // 3.判断共享数据是否到了末尾，如果到了末尾  
                if (ticket == 100) {  
                    break;  
                } else {  
                    // 4.判断共享数据是否到了末尾，如果没有到末尾  
                    try {  
                        Thread.sleep(100);  
                    } catch (InterruptedException e) {  
                        throw new RuntimeException(e);  
                    }  
                    ticket++;  
                    System.out.println(Thread.currentThread().getName() + "正在售卖第" +  ticket + "张票！");  
                }  
            }  
        }  
    }  
}

// 由于测试类中线程用的方法都是同一个因此不用给ticket加static修饰
static void main() throws InterruptedException {  
        MyRunnable myCallable = new MyRunnable();  
          
        Thread t1 = new Thread(myCallable);  
        Thread t2 = new Thread(myCallable);  
        Thread t3 = new Thread(myCallable);  
  
        t1.setName("窗口1");  
        t2.setName("窗口2");  
        t3.setName("窗口3");  
          
        t1.start();  
        t2.start();  
        t3.start();  
    }
```
**细节：** 
1. `synchronized(lock)` 不可以写在循环的外面，否则直到该线程的循环执行完之后CPU才可能被其他线程占用。
2. 锁对象必须唯一，若不唯一那么这个锁对于其他线程就没有意义，就锁不到其他不同锁的线程进入。【一般锁对象用当前类的字节码对象，如 `MyThread.class` 】
#### 同步方法：

![[Draven_Note_Images/Pasted image 20260606163001.png]]

``` java
// 由于每个Thread的参数Runable都是同一个，所有锁也是唯一的
public class MyRunnable implements Runnable {  
    // 表示这个类所有的对象，都共享ticket数据  
    int ticket = 0;  
    @Override  
    public void run() {  
        // 1.循环  
        while (true) {  
            // 2.同步代码块（同步方法）  
            if (method()) break;  
        }  
    }  
  
    private synchronized boolean method() {  
        // 3.判断共享数据是否到了末尾，如果到了末尾  
        if (ticket == 100) {  
            return true;  
        } else {  
            // 4.判断共享数据是否到了末尾，如果没有到末尾  
            try {  
                Thread.sleep(100);  
            } catch (InterruptedException e) {  
                throw new RuntimeException(e);  
            }  
            ticket++;  
            System.out.println(Thread.currentThread().getName() + "正在售卖第" +  ticket + "张票！");  
        }  
        return false;  
    }  
}
```

**举例：** `StringBuffer` 就是 `StringBuilder` 在方法上加了 `synchronized` 的版本，这样在多线程中就更加的安全。

#### `Lock` 锁：

##### 概念：

![[Draven_Note_Images/Pasted image 20260606184820.png]]

``` java
public class MyThread extends Thread{  
    // 表示这个类所有的对象，都共享ticket数据  
    static int ticket = 0;  
  
    // 创建锁对象, 注意加static确保锁的唯一  
    static Lock lock = new ReentrantLock();  
  
    public MyThread(String name) {  
        super(name);  
    }  
    @Override  
    public void run() {  
        while (true) {  
            lock.lock();  
            try {  
                if (ticket < 100) {  
                    Thread.sleep(100);  
                    ticket++;  
                    System.out.println(getName() + "正在售卖第" +  ticket + "张票！");  
                } else {  
                    break;  
                }  
            } catch (InterruptedException e) {  
                throw new RuntimeException(e);  
            } finally {  
                // 防止出现执行完成后锁未释放的问题  
                lock.unlock();  
            }  
        }  
    }  
}
```

#### 死锁

##### 概念：

死锁是多个线程互相占用对方所需的同步锁资源，彼此等待对方释放锁，导致所有线程永久阻塞、程序停滞的现象。
产生死锁需同时满足四个条件：
1. 资源互斥使用
2. 线程占有已有资源再申请新资源
3. 资源不能被强行剥夺
4. 线程形成循环等待资源的环路。

##### 举例：

![[Draven_Note_Images/Pasted image 20260606190907.png]]

``` java
public class MyThread extends Thread {  
    static Object objA = new Object();  
    static Object objB = new Object();  
    @Override  
    public void run() {  
        //1.循环  
        while (true) {  
            if ("线程A".equals(getName())) {  
                synchronized (objA) {  
                    System.out.println("线程A拿到了A锁，准备拿B锁");  
                    synchronized (objB) {  
                        System.out.println("线程A拿到了B锁，顺利执行完一轮");  
                    }  
                }  
            } else if ("线程B".equals(getName())) {  
                if ("线程B".equals(getName())) {  
                    synchronized (objB) {  
                        System.out.println("线程B拿到了B锁，准备拿A锁");  
                        synchronized (objA) {  
                            System.out.println("线程B拿到了A锁，顺利执行完一轮");
                        }  
                    }  
                }  
            }  
        }  
    }  
}
```

# 生产者和消费者
**注：** 又称为等待唤醒机制。

![[Draven_Note_Images/Pasted image 20260606191355.png]]
### 情况分类：

#### 生产者和消费者完整执行机制：

![[Draven_Note_Images/Pasted image 20260606192519.png]]

#### 情况1：理想情况

生产者先抢到了CPU执行权，此时桌子上没有食物，生产者做出了生产之后消费者线程来吃，然后依次占用线程。

#### 情况2：消费者等待

![[Draven_Note_Images/Pasted image 20260606192339.png]]

消费者先抢到CPU线程，判断桌子上有没有食物，若没有食物则消费者等待，此时生产者就能抢到线程，然后生成并唤醒消费者，随后消费者吃，吃完之后二者又进入抢占线程的状态。

#### 情况3：生产者等待

![[Draven_Note_Images/Pasted image 20260606192426.png]]

生产者先抢到了CPU执行权，判断桌子上有没有食物，若没有食物则进行生产，生产结束后唤醒消费者，若生产者又抢到了CPU执行权，发现桌子上还有食物，则生产者进行等待，此时消费者就能顺利抢到CPU执行权

### 代码实现：

#### 媒介代码：

``` java
public class Desk {  
  
    /**  
     * 作用：控制生产者和消费者的执行  
     */  
  
    // 表示桌子上是否有面条：0-无 1-有  
    public static int foodFlag = 0;  
  
    // 表示消费者可以消费的总个数  
    public static int count = 10;  
  
    // 锁对象  
    public static Object lock = new Object();  
}
```

#### 消费者代码：

**注意：** 用同一个对象的 `wait()` 方法能让它在后面 `notify` 的时候可以找到锁绑定该对象的成员。
``` java
public class Foodie extends Thread{  
    @Override  
    public void run() {  
        while (true) {  
            synchronized (Desk.lock) {  
                if (Desk.count == 0) {  
                    break;  
                } else {  
                    // 判断桌子上是否有食物  
                    if (Desk.foodFlag == 0) {  
                        // 如果没有，就等待  
                        try {  
                            // 让当前线程与锁进行绑定  
                            Desk.lock.wait();  
                        } catch (InterruptedException e) {  
                            throw new RuntimeException(e);  
                        }  
                    } else {  
                        // 吃的总数-1  
                        Desk.count--;  
                        // 如过有，就开吃  
                        System.out.println("吃货在吃面条，还能再吃" + Desk.count + "碗！！！");  
                        // 修改桌子的状态  
                        Desk.foodFlag = 0;  
                        // 唤醒厨师  
                        Desk.lock.notifyAll();  
                    }  
                }  
            }  
        }  
    }  
}
```

#### 生产者代码：

``` java
public class Cook extends Thread {  
    @Override  
    public void run() {  
        while (true) {  
            synchronized (Desk.lock) {  
                if (Desk.count == 0) {  
                    break;  
                } else {  
                    // 判断桌子上是否有食物  
                    if (Desk.foodFlag == 1) {  
                        // 如果有，就等待  
                        try {  
                            // 让当前线程与锁进行绑定  
                            Desk.lock.wait();  
                        } catch (InterruptedException e) {  
                            throw new RuntimeException(e);  
                        }  
                    } else {  
                        // 如过没有，就生产食物  
                        System.out.println("厨师做了一碗面条");  
                        // 修改桌子的状态  
                        Desk.foodFlag = 1;  
                        // 唤醒吃货  
                        Desk.lock.notifyAll();  
                    }  
                }  
            }  
        }  
    }  
}
```

#### 测试代码：

``` java
public class Test {  
    static void main() {  
        // 创建线程  
        Cook cook = new Cook();  
        Foodie foodie = new Foodie();  
  
        // 设置线程名称  
        cook.setName("厨师");  
        foodie.setName("吃货");  
  
        // 开启线程  
        cook.start();  
        foodie.start();  
    }  
}
```

### 阻塞队列方式：

#### 概念：

![[Draven_Note_Images/Pasted image 20260606195330.png]]
**注意：** 阻塞队列底层拥有锁机制。

#### 继承结构：

![[Draven_Note_Images/Pasted image 20260606195603.png]]

#### 代码实现：

`Cook` 类代码
``` java
public class Cook extends Thread {  
    ArrayBlockingQueue<String> arrayBlockingQueue;  
  
    public Cook(ArrayBlockingQueue<String> arrayBlockingQueue) {  
        this.arrayBlockingQueue = arrayBlockingQueue;  
    }  
  
    @Override  
    public void run() {  
        while (true) {  
            // 不断的把面条放到阻塞队列中  
            try {  
                arrayBlockingQueue.put("面条");  
                System.out.println("厨师放了一碗面条");  
            } catch (InterruptedException e) {  
                throw new RuntimeException(e);  
            }  
        }  
    }  
}
```

`Foodie` 类代码
``` java
public class Foodie extends Thread {  
    ArrayBlockingQueue<String> arrayBlockingQueue;  
  
    public Foodie(ArrayBlockingQueue<String> arrayBlockingQueue) {  
        this.arrayBlockingQueue = arrayBlockingQueue;  
    }  
  
    @Override  
    public void run() {  
        while (true) {  
            // 不断从阻塞队列中获取面条  
            try {  
                String food = arrayBlockingQueue.take();  
                System.out.println(food);  
            } catch (InterruptedException e) {  
                throw new RuntimeException(e);  
            }  
        }  
    }  
}
```

测试类代码
``` java
public class Test {  
  
    /**  
     * 需求：利用阻塞队列完成生产者和消费者（等待唤醒机制）的代码  
     * 细节：生产者和消费者必须使用同一个阻塞队列  
     */  
  
    static void main() {  
        // 1.创建阻塞队列的对象  
        ArrayBlockingQueue<String> arrayBlockingQueue = new ArrayBlockingQueue<>(1);  
  
        // 2.创建线程的对象，并把阻塞队列传递过去  
        Cook cook = new Cook(arrayBlockingQueue);  
        Foodie foodie = new Foodie(arrayBlockingQueue);  
  
        // 3.开启线程  
        cook.start();  
        foodie.start();  
    }  
}
```

#### 缺点：

阻塞队列里面虽然配置了锁，但是这个锁无法保护阻塞队列外的操作，虽然结果是对的，但是阻塞队列外的执行会因为线程占用CPU的随机性而混乱。

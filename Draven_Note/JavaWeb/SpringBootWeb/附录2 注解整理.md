#### `@SpringBootConfiguration`：

``` 
    标记该类是一个配置类，其中可能包含用 **`@Bean`** 注解标记的方法，用于定义 **`Bean`** 对象。
```

***

#### `@EnableAutoConfiguration`：

```
    通过启用 **`Spring Boot`** 的自动配置机制，根据项目的依赖和配置信息来自动配置应用程序。
```

***

#### `@ComponentScan`：

```
    扫描指定包及其子包下的组件（包括 **`@Component`**、**`@Service`**、**`@Repository `**等注解标记的类），将它们注册为 **`Spring`** 的组件。
```

这个组合注解简化了 **`Spring Boot`** 应用的配置，可以减少开发者的工作量。通常情况下，我们将 **`@SpringBootApplication`** 注解放置在项目的主类上。

```java
@SpringBootApplication
public class MyApp {
    public static void main(String[] args) {
        SpringApplication.run(MyApp.class, args);
    }
}
```

***

#### `@Bean`：

**`@Bean`** 是 **`Spring Framework`** 中的一个注解，用于告诉 **`Spring`** 容器将被注解标记的方法的返回值作为一个 **`Bean`** 注册到容器中。

```
    通常情况下，我们需要在 **`Spring`** 容器中创建和管理对象。通过 **`@Bean`** 注解，我们可以自定义方法来返回或创建这些对象，并将它们注册为 **`Spring`** 的 **`Bean`**，使得它们可以在其他地方进行依赖注入、使用和管理。
```

标准用法：**`@Configuration`** 配置类中的方法（推荐）

```java
@Configuration
public class AppConfig {
    @Bean
    public UserService userService() {
        return new UserServiceImpl();
    }
}
```

***

#### 元注解：

* **`@Target`：**这个注解**能写在哪**（类 / 方法 / 字段 / 参数…）
* **`@Retention`：**这个注解**活多久**（源码 / 编译后 / 运行时）
* **`@Documented`：**生成文档时显示
* **`@Inherited`：**子类可以继承父类上的该注解

```java
// 1. 元注解：用来约束【这个注解本身】的行为
@元注解1(参数)
@元注解2(参数)

// 2. 定义注解类型：固定用 @interface
public @interface 注解名 {

    // 3. 注解属性（像无参抽象方法）
    类型 属性名();

    // 带默认值的属性
    类型 属性名() default 默认值;
}
```

***

#### `@ConfigurationProperties()`：

`prefix`：看配置文件中是否有该属性。

***

#### `@EnableConfigurationProperties`：

**`@EnableConfigurationProperties`** 是 **`Spring Boot`** 提供的核心注解，用于启用并注册被**`@ConfigurationProperties`** 注解标记的配置类为 **`Spring Bean`**，实现配置文件属性与 Java 对象的绑定。

**核心职责：**激活配置属性绑定功能，将指定的配置类纳入 **`Spring`** 容器管理

**解决场景：**当配置类未被组件扫描（如第三方包、独立模块）或未添加 **`@Component`** 等注解时，通过此注解手动启用配置绑定

**底层原理：**通过 **`@Import`** 导入 **`EnableConfigurationPropertiesRegistrar`** 类，完成配置类的注册与属性绑定。

***

#### 条件注解\[`conditional`]：

1. **顶层父注解**：`@Conditional`（Spring 原生，核心基类）
2. **SpringBoot 衍生常用注解**：\
   `@ConditionalOnBean`、`@ConditionalOnMissingBean`、`@ConditionalOnClass`、`@ConditionalOnMissingClass`
3. 所有衍生注解**底层都继承/依赖** `@Conditional` 机制，实现条件化 Bean 加载。

| **注解名称** | **核心作用** | **底层父类/原理** | **关键注意事项** | **典型使用场景** |
| :---: | --- | --- | --- | --- |
| **`@Conditional`** | 自定义条件，**满足条件才注册`Bean`** | **所有条件注解的顶层父注解**   需自定义 `Condition` 接口实现判断逻辑 | 1. 原生注解，需手动写判断类   2. 粒度灵活，代码复杂度高   3. 优先级低于Boot衍生注解 | 自定义复杂条件判断、多条件组合过滤Bean |
| **`@ConditionalOnBean`** | **容器中存在指定`Bean`**时，才创建当前`Bean` | 基于 `@Conditional` 封装   内置Bean存在判断逻辑 | 1. 依赖Bean的**加载顺序**   2. 避免循环依赖   3. 只判断Bean实例，非类 | 依赖框架组件、必须依赖某个Bean才启用功能 |
| **`@ConditionalOnMissingBean`** | **容器中无指定`Bean`**时，才创建当前`Bean` | 基于 `@Conditional` 封装   反向Bean判断 | 1. **SpringBoot自动配置核心**   2. 自定义Bean优先级 > 框架默认Bean   3. 可指定类型/Bean名称匹配 | 提供默认实现、兜底Bean、允许用户自定义覆盖 |
| **`@ConditionalOnClass`** | **classpath下存在指定类**时，才创建`Bean` | 基于 `@Conditional` 封装   类加载判断 | 1. 只判断**类是否存在**，不判断实例   2. 防止缺失依赖导致报错 | 依赖第三方Jar（Jackson/MyBatis）时才加载配置 |
| **`@ConditionalOnMissingClass`** | **classpath下无指定类**时，才创建`Bean` | 基于 `@Conditional` 封装   反向类判断 | 1. 多用于降级方案   2. 避免多依赖冲突 | 无第三方依赖时，启用简易兜底配置 |

* `@ConditionalOnMissingBean`：**默认配置兜底**，SpringBoot 自动配置灵魂；
* `@ConditionalOnClass`：**依赖锁**，防止无Jar包报错；
* `@ConditionalOnBean`：**依赖绑定**，组件联动加载。

**极简记忆**

1. `@Conditional`：**父类、自定义条件**
2. `OnBean`：**有Bean才开**
3. `OnMissingBean`：**没Bean才开（默认）**
4. `OnClass`：**有类才开**
5. `OnMissingClass`：**没类才开**

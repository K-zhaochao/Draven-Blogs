### 线程池概念：

![img](/Draven_Note_Images/Pasted%20image%2020260607104130.png)

**思路：** 当出现任务的时候，线程池分配线程给该任务，该任务完成之后线程返回线程池，下一个任务来的时候又可以用该线程，若出现的任务数量大于已有的线程数量，线程池又会创建新的线程。且可设置最大线程，前面的线程先用线程池已有的最大线程数目，后面的任务则排队等待。

### 线程池的实现：

![img](/Draven_Note_Images/Pasted%20image%2020260607104501.png)

`MyRunnable` 类代码
``` java
public class MyRunnable implements Runnable {
    @Override
    public void run() {  
        for(int i=0;i<100;i++){
            System.out.println(Thread.currentThread().getName()+"-"+i);
        }  
    }  
}
```

测试类代码
``` java
public class Test {  
    static void main() {  
        // 1.获取线程池对象
        ExecutorService pool1 = Executors.newCachedThreadPool();
  
        // 2.提交任务
        pool1.submit(new MyRunnable());
  
        // 3.销毁线程池
        pool1.shutdown();
    }  
}
```

### 自定义线程池：

#### 线程池类：

![img](/Draven_Note_Images/Pasted%20image%2020260607111109.png)
**注：** `ThreadPoolExecutor` 类的构造方法参数意义。

##### 细节1：

当 `任务数量 > 核心线程数量 + 队列长度` 时，才会开启临时线程去接待多出来的任务。
![img](/Draven_Note_Images/Pasted%20image%2020260607105946.png)

##### 细节2：

当 `任务数量 > 核心线程数量 + 临时线程 + 队列长度` 时，将会触发线程池的任务拒绝策略。
![img](/Draven_Note_Images/Pasted%20image%2020260607110122.png)

#### 任务拒绝策略：

![img](/Draven_Note_Images/Pasted%20image%2020260607110804.png)
**注意：** 第三种策略指的等待时间最久是指从入阻塞队列到抛弃这个过程中等待的时间最久，因此最先进入队列的任务反而是等待最久的任务。

#### 代码实现：

``` java
public class Test {  
    static void main() {  
        ThreadPoolExecutor pool = new ThreadPoolExecutor(  
                3, // 核心线程数量，能小于0  
                6, // 最大线程数，不能小于0，最大数量>=核心线程数量  
                60, // 空闲线程最大存活时间  
                TimeUnit.SECONDS, // 时间单位  
                new ArrayBlockingQueue<>(3),// 任务队列  
                Executors.defaultThreadFactory(), // 创建线程工厂  
                new ThreadPoolExecutor.AbortPolicy() // 任务的拒绝策略  
        );  
  
		// 提交任务
        pool.submit(new MyRunnable());  
    }  
}
```

#### CPU密集型：

当项目计算比较多，但是对硬盘或数据库的读写较少的时候用该策略。

#### I/O密集型：

读取硬盘或数据库的操作较多的时候用该策略。

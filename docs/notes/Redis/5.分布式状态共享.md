# 分布式状态共享

### 1. 核心原理

将分散在多台 Tomcat 服务器本地的 Session 集中到公共的 Redis 中。在 Java 中，我们通常使用一个全局唯一的 `UUID`（Token）作为 Redis 的 Key，把用户信息作为 Value。

### 2. Java 代码实现（Spring Boot）

在实际开发中，我们通常会在登录拦截器（Interceptor）中去操作 Redis。
``` java
@Component
public class LoginInterceptor implements HandlerInterceptor {

    @Autowired
    private StringRedisTemplate stringRedisTemplate;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        // 1. 从请求头中获取 Token（前端传过来的钥匙）
        String token = request.getHeader("authorization");
        if (StrUtil.isBlank(token)) {
            response.setStatus(401); // 未登录
            return false;
        }

        // 2. 基于 Token 查询 Redis 中的用户信息
        String key = "login:token:" + token;
        String userJson = stringRedisTemplate.opsForValue().get(key);
        
        // 3. 判断用户是否存在
        if (StrUtil.isBlank(userJson)) {
            response.setStatus(401);
            return false;
        }

        // 4. 存在，把 JSON 恢复成 Java 对象，并存入 ThreadLocal 方便后续 Controller 使用
        UserDTO user = JSONUtil.toBean(userJson, UserDTO.class);
        UserHolder.saveUser(user);

        // 5. 💡核心注意点：刷新 Token 的有效期（只要用户在操作，就不能让他掉线）
        stringRedisTemplate.expire(key, 30, TimeUnit.MINUTES);

        return true;
    }
}
```

### 注意点：

- **别忘了刷新有效期：** 用户如果连续玩了 31 分钟手机，期间一直在点餐，如果不用 `expire` 刷新时间，第 30 分钟时由于 Redis Key 到期，用户会莫名其妙被强制登出。
    
- **使用 StringRedisTemplate：** 不要用标准的 `RedisTemplate`，因为标准的默认使用 JDK 序列化，存进去的内容全是可读性极差的乱码。`StringRedisTemplate` 存进去是纯文本 JSON，方便排查问题，且省内存。
